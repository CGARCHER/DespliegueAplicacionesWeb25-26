# Práctica de Investigación: Arquitectura y Tecnologías de una Plataforma Web de Gran Escala

## Objetivo
El objetivo de esta práctica es analizar en profundidad el stack tecnológico y la arquitectura de software de una plataforma web moderna de gran escala. A través de este análisis se aplicarán conceptos relacionados con arquitecturas web, tecnologías Front-end y Back-end, bases de datos, comunicación entre servicios, despliegue y escalabilidad.

## Introducción
Las principales plataformas tecnológicas gestionan millones de usuarios, grandes volúmenes de datos y operaciones en tiempo real. Para dar soporte a estas necesidades se utilizan arquitecturas complejas, distribuidas y altamente escalables basadas en múltiples tecnologías especializadas.  
Esta práctica tiene como finalidad comprender cómo se diseñan y operan dichas infraestructuras, tomando como referencia el análisis realizado en clase sobre Netflix y trasladándolo a otra plataforma seleccionada.

## Selección de la Plataforma

- YouTube
- Instagram
- Twitter/X

- Airbnb
- Facebook
- **otra plataforma similar que te sea de interés**

## Tareas a Realizar

### 1. Modelo Arquitectónico General
- Describir el tipo de arquitectura utilizada (microservicios, monolítica, orientada a eventos, serverless, híbrida, etc.).
- Explicar por qué este modelo resulta adecuado para la escala y naturaleza del servicio.
- Analizar aspectos como escalabilidad, resiliencia, disponibilidad y mantenimiento.

### 2. Tecnologías Front-end
- Identificar los lenguajes, frameworks y librerías empleados en las distintas interfaces de usuario (web, móvil, escritorio o Smart TV).
- Explicar cómo estas tecnologías contribuyen a la experiencia de usuario en términos de interactividad, rendimiento y diseño.

### 3. Tecnologías Back-end
- Describir los lenguajes de programación utilizados en el servidor.
- Identificar los frameworks, servicios clave o motores necesarios para funcionalidades como streaming, recomendaciones, procesamiento de datos o gestión de usuarios.
- Explicar cómo estas herramientas permiten manejar grandes cargas de trabajo y tráfico global.

### 4. Sistemas de Bases de Datos y Almacenamiento
- Identificar las bases de datos relacionales, NoSQL, sistemas de caché y mecanismos de almacenamiento utilizados.
- Explicar qué tipo de información gestiona cada sistema y por qué resulta adecuado para ese propósito.

### 5. Comunicación entre Servicios y APIs
- Describir los protocolos de comunicación utilizados (REST, gRPC, GraphQL, WebSockets).
- Identificar los sistemas de mensajería o eventos (Kafka, RabbitMQ, Pub/Sub).
- Explicar el papel de las APIs internas y externas dentro de la arquitectura.

### 6. Infraestructura, Escalabilidad y Despliegue
- Identificar la plataforma de nube utilizada (AWS, Google Cloud, Azure u otra).
- Describir el uso de contenedores (Docker), orquestación (Kubernetes) y CDNs.
- Explicar las estrategias de despliegue continuo y los mecanismos de escalado automático.

### 7. Diagrama Arquitectónico
- Elaborar un diagrama propio o seleccionar uno fiable, citando la fuente correspondiente.
- El diagrama debe incluir clientes, servicios Front-end y Back-end, bases de datos, sistemas de mensajería y componentes de infraestructura.
- Incluir una explicación del flujo general de una petición desde el cliente hasta los componentes internos.
